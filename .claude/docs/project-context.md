# Petlog Project Context

## Goal

Petlog는 상용화를 목표로 하는 반려동물 건강 기록 서비스이다.

동시에 풀스택/프로덕트 엔지니어 이직을 위한 포트폴리오이기도 하다.

단순 CRUD 구현이 목적이 아니다.

상용화까지의 전 과정을 포트폴리오로 활용한다:

- 사용자 문제 정의
- 제품 설계 및 수익 모델
- 도메인 모델링
- 확장 가능한 아키텍처
- AI 기능 확장 구조
- 실제 출시 및 사용자 반응


## Success Criteria

실사용자가 실제로 쓰는 서비스를 만드는 것이 목표다.

수익보다 사용자 경험과 리텐션을 우선한다.


## MVP Goal

웹 우선 모바일 경험 제공.

핵심 사용자 흐름:

1. 사용자 가입
2. 반려동물 등록
3. 건강 기록 작성
4. 건강 변화 확인
5. AI 기반 건강 요약 확인 (유료)


## Development Scope

현재:

- Mock AI (구조 검증용)
- PostgreSQL
- NestJS API
- Next.js Frontend

Later:

- OpenAI Fine-tuned 모델 연동
- 모바일 앱 출시


## Non Goal

초기 버전에서 하지 않는다.

- 의료 진단
- 수의사 대체
- 복잡한 커뮤니티
- 과도한 AI 기능
- 보험 연동
