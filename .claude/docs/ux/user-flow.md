# Petlog 사용자 흐름

## 1. 핵심 사용자 흐름

Petlog MVP의 핵심 사용자 흐름은 다음과 같다.

```
회원가입 / 로그인
↓
반려동물 등록
↓
건강 기록 작성
↓
건강 변화 확인
↓
AI 건강 리포트 확인
```

---

## 2. 인증 흐름

### 신규 사용자

```
앱 진입
↓
회원가입 화면
↓
이메일 / 이름 / 비밀번호 입력
↓
가입 완료
↓
반려동물 등록 화면으로 이동
```

### 기존 사용자

```
앱 진입
↓
로그인 화면
↓
이메일 / 비밀번호 입력
↓
홈 화면으로 이동
```

---

## 3. 반려동물 등록 흐름

```
홈 화면 (반려동물 없음)
↓
반려동물 추가 버튼
↓
이름 / 종류 / 품종 / 생년월일 / 성별 / 몸무게 입력
↓
등록 완료
↓
반려동물 홈 화면
```

---

## 4. 건강 기록 작성 흐름

### 빠른 기록 (핵심 흐름)

```
홈 화면
↓
기록 추가 버튼
↓
기록 종류 선택 (몸무게 / 식욕 / 활동량 / 증상)
↓
값 입력
↓
메모 입력 (선택)
↓
저장
↓
홈 화면으로 복귀
```

UX 목표: 3번 탭 이내에 기록 완료.

---

## 5. 건강 타임라인 확인 흐름

```
홈 화면
↓
반려동물 선택
↓
건강 타임라인 화면
↓
날짜별 기록 확인
↓
필터: 기간 / 기록 종류 선택
↓
변화 그래프 확인
```

---

## 6. 병원 기록 흐름

```
홈 화면
↓
병원 기록 탭
↓
기록 추가 버튼
↓
병원명 / 방문 날짜 / 진료 내용 입력
↓
영수증 또는 처방전 이미지 첨부 (선택)
↓
저장
```

---

## 7. AI 리포트 흐름

```
홈 화면
↓
리포트 탭
↓
리포트 생성 버튼
↓
분석 기간 선택 (주간 / 월간)
↓
생성 중 로딩
↓
리포트 결과 확인
  - 건강 변화 요약
  - 주요 기록 하이라이트
  - 보호자 주의 사항
```

---

## 8. 다중 반려동물 관리 흐름

```
홈 화면
↓
반려동물 스위처
↓
다른 반려동물 선택
↓
선택된 반려동물의 건강 데이터 표시
```

---

## 9. 에러 흐름

### 네트워크 오류

```
API 요청 실패
↓
에러 메시지 표시
↓
재시도 버튼 제공
```

### 빈 상태

```
건강 기록 없음
↓
빈 상태 메시지 표시
↓
첫 기록 추가 유도 CTA
```

### AI 리포트 생성 실패

```
AI 생성 실패
↓
실패 안내 메시지
↓
재시도 또는 나중에 다시 시도 안내
```

---

## 10. UX 원칙

### 빠른 기록

보호자가 기록을 빠르게 남길 수 있어야 한다.

최소 탭 수로 핵심 기록 완료.

### 명확한 피드백

모든 비동기 작업에 로딩 상태를 표시한다.

성공 / 실패 여부를 명확하게 알린다.

### 다음 행동 안내

에러 상황에서도 사용자가 무엇을 해야 하는지 알 수 있어야 한다.
