---

## 평가 기준

### 1. 설계 의도 명확성
- 왜 이 구조를 선택했는지 코드에서 읽히는가?
- 도메인 모델이 코드 구조에 반영되어 있는가?
- 계층 분리가 명확한가?

### 2. 유지보수성
- 6개월 후 다른 개발자가 이 코드를 이해할 수 있는가?
- 과도한 추상화 또는 반대로 중복이 없는가?
- TypeScript 타입이 문서 역할을 하는가?

### 3. 확장 가능성
- AI Provider 교체 시 비즈니스 로직 수정 없이 가능한가?
- 새 도메인 추가 시 기존 코드 수정 범위가 최소인가?

### 4. 프로덕트 엔지니어링 관점
- 사용자 경험을 고려한 에러 처리가 있는가?
- 로딩/에러/빈 상태가 모두 처리되는가?
- 모바일 우선 UI 원칙을 지키는가?

### 5. 포트폴리오 어필 포인트
- 이 코드로 면접에서 설명할 수 있는 **기술적 결정**은 무엇인가?
- 단순 CRUD를 넘어 어떤 공학적 문제를 해결했는가?
